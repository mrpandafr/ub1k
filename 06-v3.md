# 06 — V3: Time as a Dimension

> *Timestamped consultations. The third dimension turns static space into living memory.*

---

## 6.1 The missing dimension

V2 solved the relevance problem: 10/10 correct hits. But it couldn't answer temporal questions:

- "What was I discussing at this time yesterday?"
- "How many times has this concept been accessed?"
- "What's the concept of the moment (last 5 minutes)?"
- "What concepts have I abandoned?"

These questions require an **extra dimension**, orthogonal to the semantic one.

---

## 6.2 The data

Each atom carries:

```json
{
  "at": "2026-06-29T19:04:00Z",         // creation timestamp
  "seen": [                               // consultation history
    "2026-07-09T08:30:00Z",
    "2026-07-09T08:31:00Z",
    "2026-07-09T08:32:00Z"
  ],
  "text": "Pierre Biguinet était le parrain de Nicolas.",
  "vector": [0.12, 0.34, ...],
  "bank_id": "kage",
  "tags": ["bank:kage", "role:user"]
}
```

**`at`** — when the atom was created (present since V1).  
**`seen`** — when the atom was retrieved by kNN (new in V3).

---

## 6.3 What the `seen` array enables

| Question | Mechanism |
|:---------|:----------|
| "What's hot right now?" | `seen` filtered by `now - 5min`, count frequency |
| "What did I explore yesterday at 3pm?" | `seen` filtered by `yesterday 15:00 ± 1h` |
| "Is this concept dead?" | `seen` not updated in 30+ days → archive |
| "What's my learning rhythm?" | Distribution of `seen` over hours/days |
| "How many times have I revisited Pierre?" | Length of `seen` array for Pierre atoms |

---

## 6.4 Architecture V3

```
Public-memory_atoms
┌─────────────────────────────────────────┐
│  text: string (atomic content)          │
│  vector: float[384] (semantic)          │
│  at: ISO timestamp (creation)           │
│  seen: [ISO timestamps] (consultations) │  ← new
│  bank_id: string (agent namespace)      │
│  tags: [string] (metadata)              │
└─────────────────────────────────────────┘

Query flow:
1. kNN returns top 3 atoms (11ms)
2. For each hit: append `now` to `seen` array (1ms per update)
3. Temporal queries: ES `range` filter on the `seen` array
