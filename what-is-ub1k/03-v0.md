# 03 — V0: Zero State

> *8 June 2026. 71,000 messages in SQLite. No index. No filter. No search.*

---

## 3.1 The file

A single SQLite database — **state.db** — 1.1 GB, 539 sessions, 71,175 messages. Every conversation from day one, stored in its raw form.

| Property | Value |
|:---------|:------|
| Format | SQLite |
| Size | 1.1 GB |
| Sessions | 539 |
| Messages | 71,175 |
| Search | FTS5 (full-text, not semantic) |
| TTL | None |

---

## 3.2 The problem

Every turn, the entire context was injected:

```
<memory-context>
  Session 29 juin (426 messages, including 6x chat.md)
  Session 28 juin (1722 messages with all tool calls)
  Session 18 juin (1849 messages — Sylvain, GitHub)
  ... 50,000 tokens of raw conversation
</memory-context>
```

50,000 tokens per turn, of which **90% was noise**: tool outputs, file listings, search results, error messages, context compactions.

---

## 3.3 What it felt like

The agent couldn't find anything. Not because the data wasn't there — because the signal was buried under  50x more noise. Every query for "Pierre" returned a vector that had been averaged across 200 unrelated messages.

---

## 3.4 What we needed

- **Atomicity:** One concept per embedding
- **Noise filtering:** Don't index tool calls, error messages, or empty results
- **Queryable memory:** Sub-50ms kNN, not 50k-token context stuffing
- **Temporal awareness:** Know when facts were created
