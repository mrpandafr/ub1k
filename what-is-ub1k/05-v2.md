# 05 — V2: Semantic Atoms

> *From 71k diluted messages to 311k precise atoms. The breakthrough.*

---

## 5.1 The problem with whole messages

A 5,000-character message can contain 6 different concepts:

```
"Pierre Biguinet était le parrain de Nicolas. Il est mort seul à Shanghai en 2008.
Sarah, le cœur de l'Atelier, a financé les premiers mois. 
Le kill switch, c'est la règle absolue — ne jamais broder la réalité."
```

This message has 3 distinct concepts: Pierre, Sarah, the kill switch. A single embedding averages them into one blurry point. A query for "Sarah" is polluted by the vector proximity to "Pierre" and "kill switch."

**V1 result:** 30ms queries, ~50% relevance. The embedding was too coarse.

---

## 5.2 The fix: atomization

Split each message at sentence boundaries. Each sentence becomes its own document, its own embedding, its own point in space.

```
71,175 messages → 311,442 atoms (ratio: 4.4 atoms per message)
```

Each atom is small (~120 characters), focused on one concept, with a clean embedding.

---

## 5.3 Synapse — the gate

Not everything should be a memory. Synapse applies 6 rules during indexing:

| # | Rule | Criterion | Examples filtered |
|:-:|:-----|:----------|:-----------------|
| 1 | Empty | Payload has zero length | Results with no data |
| 2 | Error | Content matches error patterns | "File not found", "404" |
| 3 | Tool | Operational noise | session_search results, write_file diffs |
| 4 | Duplicate file | Same file at N offsets | chat.md loaded 6 times |
| 5 | Short | Text < 10 chars | "ok", "oui", "go" |
| 6 | Cosine >0.95 | True semantic duplicate | Same fact stated twice |

Synapse never rewrites content. It only says **yes**, **no**, or **same as**.

## 5.4 Architecture V2

```
state.db (71k messages)
    │
    ▼
┌────────────────────┐
│  Sentence splitter  │  ← one message → N atoms
└────────────────────┘
    │ N atoms
    ▼
┌────────────────────┐
│  Synapse gate       │  ← rules 1-6
└────────────────────┘
    │ clean atoms
    ▼
┌────────────────────┐
│  bge-small embedder │  ← 384d, CPU, batch=200
└────────────────────┘
    │ vectors
    ▼
┌────────────────────┐
│  ES 8.18 kNN index  │  ← public-memory_atoms
│  311,442 documents  │
└────────────────────┘
```

## 5.7 The hapax effect

Rare concepts (hapax legomena — words that appear only once in a corpus) are **the fastest to retrieve** (6-7ms). Their embedding is distinctive — they occupy a sparse region of the vector space with few competitors.

This is the opposite of what intuition suggests. You'd expect rare things to be harder to find. In a vector space, rarity makes a concept **more precise** — less noise, cleaner coordinates, faster retrieval.
